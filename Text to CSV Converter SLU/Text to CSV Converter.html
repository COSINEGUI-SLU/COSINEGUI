Help & Documentation

This tool supports various text file formats:

  • Experimental Data: Space or tab-delimited files with m/z and intensity values
  • Predicted Data: Files with energy level sections and multiple peaks per energy level
  • Files with or without headers
  • Files with comment lines (starting with #)

Example of experimental data format:

100.0123 5000.25
101.2345 3456.78
102.3456 2345.67

Example of predicted data format:

energy0
136.06177 75.41 3 36 50 39 37 49 33 45
298.09684 100.00 0 109 112 120 139 142
energy1
134.04612 3.47 35 38
136.06177 100.00 3 33 39 50 45 36 37 49

  1. Go to the Convert tab
  2. Drag and drop your text file or click "Browse Files"
  3. Adjust settings if needed:
    • Delimiter: Choose the character that separates columns in your file
    • Normalize intensity: Scale intensity values to a maximum of 100
    • Header row: Check if your file has a header row
  4. Click "Convert to CSV"
  5. Preview the converted data
  6. Click "Download CSV" to save the result

  1. Go to the Match Data tab
  2. Upload both experimental and predicted data files
  3. Set the m/z tolerance value (default: 0.01)
  4. Click "Match Data"
  5. Review the match results
  6. Click "Download Matched Data" to save the matched datasets

The matching algorithm finds pairs of peaks from both datasets where the m/z values differ by less than the specified tolerance.

Common Issues:
  • File not parsing correctly: Try selecting a different delimiter
  • No matches found: Try increasing the m/z tolerance value
  • Browser performance issues: Large files may cause slowdowns; try splitting into smaller files
  • Download not working: Some browsers may block downloads; check your browser settings

If you encounter persistent issues, try using a different browser or contact support.
